# Overwatcheat 使用说明

> 版本 5.1.0 | 仅支持 Windows | 需要 JDK 19+

---

## 第一步：安装前置依赖

### 1. 安装 JDK 19

前往 [https://adoptium.net](https://adoptium.net) 下载并安装 **JDK 19** 或更高版本。

安装完成后，打开命令提示符验证：

```bat
java -version
```

输出中应包含 `version "19"` 或更高版本号。

### 2. 安装 Interception 驱动

Interception 是一个 Windows 内核级输入驱动，程序必须依赖它来注入鼠标移动。

1. 前往 [https://github.com/oblitum/Interception/releases](https://github.com/oblitum/Interception/releases) 下载最新版本
2. 以**管理员身份**运行安装程序
3. 安装完成后**重启电脑**

> 若未安装此驱动，程序启动时将报错退出。

---

## 第二步：构建程序

在项目根目录下，双击运行：

```
build.bat
```

等待 Gradle 下载依赖并完成编译（首次构建需要较长时间）。构建成功后，可运行的程序位于：

```
build\overwatcheat\
```

该目录包含以下文件：

```
overwatcheat\
├── overwatcheat.jar    ← 主程序
├── run.bat             ← 启动脚本
├── overwatcheat.cfg    ← 配置文件（你需要编辑此文件）
├── LICENSE.txt
└── README.md
```

---

## 第三步：配置 overwatcheat.cfg

用记事本或任意文本编辑器打开 `overwatcheat.cfg`，根据需要修改以下关键配置。

### 必须配置

**灵敏度（最重要！）**

```ini
sensitivity=10.0
```

将此值改为你在游戏内设置的**实际灵敏度**，两者必须一致，否则瞄准会过快或过慢。

**激活键**

```ini
aim_key=1
```

`1` 表示鼠标左键。若希望用其他键激活，可改为对应的 [Windows 虚拟键码](https://docs.microsoft.com/en-us/windows/win32/inputdev/virtual-key-codes)（十进制）。

> 使用甩枪模式时，**不要**将此键设为鼠标左键。

**瞄准模式**

```ini
aim_mode=0
```

| 值 | 模式 | 适用英雄 |
|----|------|----------|
| `0` | 追踪（Tracking） | 士兵76、源氏等自动武器 |
| `1` | 甩枪（Flicking） | 麦克雷、艾什、黑百合等单发英雄 |

### 常用调整

**扫描区域大小**

```ini
box_width=64
box_height=32
```

扫描范围是以屏幕中心为中心的矩形区域。数值越大，能锁定越远的目标，但误判风险也越高。4K 分辨率建议适当调大（如 128×64）。

**瞄准速度**

```ini
aim_duration_millis=0.78
```

值越小瞄准越快越"硬"，值越大越平滑。

**随机抖动（模拟人类行为）**

```ini
aim_jitter_percent=64
```

取值 0~100。`64` 表示有 64% 的概率会引入随机偏差，使行为更像人类。若设为 `0` 则完全精准。

**甩枪模式参数**（`aim_mode=1` 时生效）

```ini
flick_shoot_pixels=5
flick_pause_duration=200
```

`flick_shoot_pixels`：准星与目标距离小于此像素数时自动射击。  
`flick_pause_duration`：两次射击之间的间隔（毫秒）。不同英雄参考值：麦克雷 200、艾什 350、黑百合 270。

**Interception 设备 ID**

```ini
mouse_id=11
keyboard_id=1
```

通常保持默认即可。若有多个鼠标/键盘设备，可尝试调整（鼠标范围 11~20，键盘范围 1~10）。

**CPU 亲和性（性能优化）**

```ini
aim_cpu_thread_affinity_index=2
```

将瞄准线程绑定到指定 CPU 核心，减少调度抖动。  
- 设为 `-1` 可禁用此功能  
- 建议绑定物理核心（超线程 CPU 上为偶数编号，如 2、4、6）  
- **不要设为 0**，否则可能严重降低游戏帧率  
- 核心数少于 4 的 CPU 建议禁用（设为 `-1`）

---

## 第四步：启动程序

### 游戏设置准备

1. 以**全屏窗口模式**运行《守望先锋》
2. 确认游戏内灵敏度与 `sensitivity` 配置项一致

### 启动

进入 `build\overwatcheat\` 目录，以**管理员身份**运行：

```
run.bat
```

> 必须以管理员身份运行，否则 Interception 驱动无法正常工作。

### 使用

- **按住** `aim_key` 配置的按键（默认鼠标左键）：激活自动瞄准
- **松开按键**：停止自动瞄准
- 追踪模式下持续按住即可持续跟随目标
- 甩枪模式下程序会在准星接近目标时自动触发左键点击

---

## OBS 投影模式（颜色封禁时使用）

若游戏报 **error 5**（颜色被封禁），可改用 OBS 全屏投影窗口作为捕获源：

1. 打开 OBS，在场景预览上**右键** → **全屏投影（Fullscreen Projector）** → 选择你的显示器
2. 修改配置文件：
   ```ini
   window_title_search=Fullscreen Projector
   ```
3. 重新启动程序

## 常见问题

**Q：程序启动报错，提示找不到 interception.dll？**  
A：Interception 驱动未正确安装，请以管理员身份重新安装并重启电脑。

**Q：程序启动后瞄准完全不动？**  
A：检查 `mouse_id` 是否正确（尝试 11~20 范围内的其他值）；确认程序以管理员身份运行。

**Q：瞄准方向偏移，总是打不中目标？**  
A：`sensitivity` 配置项与游戏内灵敏度不一致，请对照游戏内设置修改。

**Q：瞄准速度太快/太慢？**  
A：调整 `aim_duration_millis`（越小越快）和 `sensitivity`（越大越慢）。

**Q：找不到游戏窗口？**  
A：确认游戏正在运行且处于全屏窗口模式；检查 `window_title_search` 值是否包含游戏窗口标题关键词。

**Q：CPU 占用过高？**  
A：降低 `fps`（如改为 30），或将 `aim_precise_sleeper_type` 改为 `2`（SLEEP 模式）。

**Q：4K 分辨率下检测不到目标？**  
A：适当增大 `box_width` 和 `box_height`（如 128×64 或 256×128）。

## 精确睡眠模式说明

`aim_precise_sleeper_type` 控制瞄准线程的等待方式，影响精度与 CPU 占用的平衡：

| 值 | 模式 | 精度 | CPU 占用 | 适用场景 |
|----|------|------|----------|----------|
| `0` | YIELD | 中 | 中 | 推荐默认 |
| `1` | SPIN_WAIT | 最高 | 高 | 追求极致精度 |
| `2` | SLEEP | 低 | 低 | CPU 性能较弱时 |
