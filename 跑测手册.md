# Overwatcheat 跑测手册

> 最后更新：2026-04-15
> 当前状态：颜色识别已通过，鼠标移动待验证

---

## 一、运行前准备

### 1. 游戏设置（每次测试前确认）

- 游戏模式：**全屏窗口**（不是全屏独占）
- 辅助功能 → 色觉辅助 → 开启**蓝色盲模式**
  - 这会把敌人血条变成 `#ff00ff` 纯品红色，程序才能识别

### 2. 运行环境

- 以**管理员权限**打开 PowerShell
- 进入项目目录：`cd d:\Overwatcheat`

---

## 二、启动程序

```powershell
.\gradlew.bat run
```

程序启动成功标志（终端无报错，屏幕左上角出现黄色调试文字）：
```
frame #xxx | 256x256 (Robot)
matched pixels: xxx | total matches: xxx
```

---

## 三、调试叠加层说明

程序运行后会在游戏画面上叠加以下信息：

| 元素 | 含义 |
|------|------|
| **黄色文字**（左上角第1行） | 帧号、分辨率、匹配像素数 |
| **橙色文字**（左上角第2行） | KEY状态、鼠标指令内容、已发送次数 |
| **青色文字**（左上角下方） | 当前帧中品红色像素的颜色采样 |
| **青色虚线框** | 256×256 扫描区域边界（屏幕中心） |
| **绿色实线框 + "已识别"** | 检测到血条，框住目标位置 |
| **红色大字 "未识别到"** | 当前帧无匹配像素 |

---

## 四、测试步骤

### 步骤 1：验证颜色识别

1. 进训练场，站到能看见机器人血条的位置
2. 把准星**大致对准机器人**（不需要精确，在屏幕中心 256×256 区域内即可）
3. 检查叠加层：
   - ✅ 出现绿框 + "已识别" → 颜色识别正常
   - ❌ 显示"未识别到" → 检查蓝色盲模式是否开启

### 步骤 2：验证按键检测

1. **按住鼠标左键**（`aim_key=1`）
2. 看橙色文字变化：
   - ✅ `KEY:DOWN` → 按键检测正常
   - ❌ 一直显示 `KEY:UP` → 按键检测有问题

### 步骤 3：验证鼠标移动指令

按住左键并对准血条，看橙色文字：

| 橙色文字内容 | 含义 | 处理方式 |
|-------------|------|----------|
| `SENT(x,y) mouseId=1` | 指令已发出 ✅ | 看鼠标实际有没有动 |
| `SKIP:minSize` | 目标像素区域太小 | 靠近敌人，或降低 `aim_min_target_width/height` |
| `SKIP:maxSnap dX=xxx` | 目标偏移超出最大捕捉范围 | 先大致对准再按键 |
| `NO_DATA` | 没有检测到目标 | 见步骤1 |

### 步骤 4：验证鼠标实际移动

如果第3步显示 `SENT` 但鼠标没动：

1. 当前 `mouse_id=1`（由 `identify.exe` 确认）
2. 如果无效，逐一尝试 `mouse_id=11` 到 `mouse_id=20`
3. 每次改完 `overwatcheat.cfg` 后**必须重启程序**

## 五、当前配置快查

```ini
aim_key=1               # 鼠标左键触发
aim_mode=0              # TRACKING 模式
sensitivity=6           # 游戏内灵敏度，必须和游戏内一致
box_width=256           # 扫描区域宽度（像素）
box_height=256          # 扫描区域高度（像素）
target_colors=ff00ff... # 蓝色盲模式血条颜色
target_color_tolerance=0 # 精确匹配，无容差
window_title_search=desktop  # 直接截屏，不查找窗口
mouse_id=1              # 由 identify.exe 确认的鼠标 ID
keyboard_id=1
aim_max_move_pixels=50  # 调试用大值，确认有效后改回 3
aim_jitter_percent=64   # 人形抖动，测试阶段可改为 0
```

> ⚠️ `aim_max_move_pixels=50` 是调试用的大值，确认鼠标移动有效后改回 `3`

## 六、常用诊断工具

| 工具 | 路径 | 用途 |
|------|------|------|
| identify.exe | `Interception/samples/x86/identify.exe` | 查看鼠标/键盘的 Interception 设备 ID |
| debug_frames/ | `d:\Overwatcheat\debug_frames\` | 程序每5秒保存的截图，确认截取的是游戏画面 |

运行 `identify.exe` 后移动鼠标，输出 `INTERCEPTION_MOUSE(N)` 中的 N 就是设备序号（对应 `mouse_id = 10 + N`，目前实测是 N=1 → 先试 id=1，再试 id=11）。

## 八、恢复正常使用配置（调试完成后）

```ini
aim_max_move_pixels=3
aim_jitter_percent=64
target_color_tolerance=0
```

然后根据实际效果微调 `sensitivity`（要和游戏内灵敏度完全一致）。
